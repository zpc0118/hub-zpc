LLM API 自动切换演示

多 Provider 故障转移 · 断路器模式 · 主动健康探针
你好！这是 LLM 故障转移（Failover） 演示。 工作原理： • 后台维护 Provider 优先级链（百炼 → DeepSeek） • 每个 Provider 有独立的断路器，追踪健康状态 • 发消息时从第一个 Provider 尝试，不可用则自动切换 三种不可用状态（右侧可模拟）： • 🔧 维护中 — 手动下线，立即跳过 • 🔴 断路器 OPEN — 累积错误后自动断路，冷却后探针恢复 • 🟡 HALF_OPEN — 冷却时间到，正在发探针试探是否恢复 点击右侧按钮体验各种场景 👉
系统提示
🔌 Provider 状态
CLOSED — 正常，放行所有请求
OPEN — 断路，快速失败；冷却后自动探针
HALF_OPEN — 探针通过则恢复，失败则重新断路
维护中 — 人工下线，不影响断路器
