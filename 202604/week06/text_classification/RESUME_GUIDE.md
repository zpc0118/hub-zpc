## 三、按岗位方向写法

### 3.1 算法工程师 / NLP 工程师

> 侧重：模型结构、训练技巧、指标结果、对比实验

**简历文字示例**：

> **中文新闻多分类系统（BERT Fine-tuning vs LLM SFT 三路对比）**
>
> - 基于 `bert-base-chinese`（110M）手写 BertModel + Linear 分类头，对比 CLS/Mean/Max 三种池化策略，在 CLUE TNEWS（53K条，15类）上验证集准确率达 **60%**，Macro F1 约 50%
> - 针对类别不均衡（最多/最少类 = 23x），引入 `compute_class_weight` 加权 CrossEntropyLoss，少数类（证券）Recall 提升约 15pp
> - 采用分层学习率（BERT 层 2e-5，分类头 1e-4）+ Linear Warmup 调度，对比统一学习率方案收敛更稳定
> - 基于 `Qwen2-0.5B-Instruct`（500M）实现 **LoRA 指令微调**：将分类任务转化为 chat 格式，实现 loss masking（只在类别 token 上计算损失），可训练参数仅 **0.22%**（1.08M/495M），用 5K 条数据训练 18 分钟达到 **58.0% 准确率**，与 BERT 全量微调（53K 条）效果相当
> - 三路对比量化：BERT fine-tune（判别式）/ LLM zero-shot（无需训练）/ LLM SFT（指令微调），从准确率、数据效率、推理延迟三维度给出选型依据

### 3.2 后端 / AI 工程化岗位

> 侧重：模块化设计、工程规范、可复用性

**简历文字示例**：

> **中文文本分类工程化项目（BERT + PyTorch）**
>
> - 将 BERT 文本分类拆分为 5 个独立模块（dataset / model / train / evaluate / predict），各模块职责清晰，可独立替换（如换 RoBERTa 只需改 `build_model` 一处）
> - 设计三种可插拔的池化策略（cls/mean/max），通过 `--pool` 参数切换，无需修改训练代码
> - 支持 GPU/CPU 自动兼容、梯度累积（`--grad_accum`）和 checkpoint 保存（仅保留最优），实现工业级训练流程
> - 完成数据探索脚本，自动生成类别分布、长度统计、截断覆盖率等 5 张分析图表

### 3.3 数据工程师 / 数据科学家

> 侧重：数据分析、实验设计、指标理解

**简历文字示例**：

> **CLUE TNEWS 新闻分类数据分析与建模**
>
> - 对 53,360 条中文新闻标题进行系统性 EDA：量化类别不均衡比（23x）、文本长度分布（均值 22 字，P99=39 字）、BERT Token/字符比值（≈0.97），数据分析结论直接驱动了截断长度（64）和加权 Loss 的选型
> - 设计消融实验矩阵：对比 3 种池化策略 × 有无加权 loss × 不同训练轮数，用 Macro F1 + 混淆矩阵量化每个设计决策的收益
> - 实现自动评估脚本，输出 per-class Precision/Recall/F1 和归一化混淆矩阵，定位高频误分类类别对

## 六、面试常见问题

**Q1：BERT 的 [CLS] token 为什么可以用来做分类？**

> 预期答法：BERT 预训练时用 NSP（Next Sentence Prediction）任务，[CLS] 被设计为汇聚整句信息的特殊 token。fine-tuning 时在其上加线性层，SGD 会进一步强化 [CLS] 的句子表示能力。

**Q2：Mean Pooling 和 CLS 哪个更好？**

> 预期答法：没有绝对答案，取决于任务。[CLS] 在分类任务中通常够用；Mean Pooling 在语义相似度（Sentence-BERT）中表现更好，因为它利用了所有 token 的信息。本项目有实验数据，引用自己的消融结果回答。

**Q3：为什么 BERT 层和分类头要用不同的学习率？**

> 预期答法：BERT 预训练权重已经包含丰富的语言知识，用大学习率会破坏（catastrophic forgetting）；分类头是随机初始化的，需要更大步长收敛。通常 BERT 层 1e-5~3e-5，分类头可以大 5~10 倍。

**Q4：类别不均衡除了加权 Loss 还有哪些处理方式？**

> 预期答法：
> 1. **过采样/欠采样**：对小类重复采样或删减大类样本
> 2. **Focal Loss**：自动降低易分类样本的权重，让模型专注于难分类样本
> 3. **数据增强**：对小类做回译、同义词替换等增强
> 4. **模型层面**：最后一层 bias 初始化为类别先验的 log-odds（适合极端不均衡）

**Q5：BERT 的 max_length 设为多少合适？这个项目你设了多少？**

> 预期答法：理论上看数据分布决定。本项目数据分析显示 P99 = 39 字，设 64 已覆盖所有样本。设 128 是安全冗余但会浪费计算资源（attention 复杂度是 O(L²)）。实际生产中要在覆盖率和效率之间权衡。

**Q6：zero-shot 效果比 fine-tuning 差很多，还有意义吗？**

> 预期答法：有。zero-shot 的价值在于无需标注数据，在数据获取成本高、类别动态变化的场景（如新闻类别频繁调整）更有优势。两种方案各有适用场景：有充足标注数据 → fine-tuning；快速验证、无标注 → zero-shot。本项目同时实现两者，目的就是让学生有选型依据。

**Q7：你的项目 Macro F1 只有 50%，是不是效果太差？**

> 预期答法：要看 baseline 和任务难度。TNEWS 任务上 BERT-base 的 CLUE 榜单参考值约 56%~58%（训练更多 epoch、更精细调参），本项目 3 epoch 基础配置达到约 50% 是合理起点。更重要的是能解释哪些类别难分（如旅游/国际/文化容易混淆），以及改进方向，这比单纯堆指标更有价值。

**Q8：LoRA 是什么原理？为什么可以只训练 0.22% 的参数？**

> 预期答法：LoRA 在原始权重矩阵 W（冻结）旁边插入低秩分解 ΔW = B·A，其中 A ∈ R^{r×d}，B ∈ R^{d×r}，r << d。以 Qwen2-0.5B 为例，attention 的 q_proj 矩阵是 896×896，全参数 80 万；LoRA r=8 只需 A（8×896）+ B（896×8）= 14336 个参数，减少 55 倍。训练时只更新 A 和 B，推理时将 BA 加回 W，不改变模型结构。

**Q9：SFT 的 loss masking 是什么？为什么要这样做？**

> 预期答法：SFT 训练时只对 assistant 回复部分（本项目是类别名 + EOS）计算 cross-entropy loss，prompt 部分（system + user）的 labels 设为 -100，PyTorch 会自动忽略这些位置。原因：如果对整个序列计算 loss，模型会花大量梯度去"记住"prompt 的格式，而我们只想让它学"给定 prompt 应该输出什么答案"。pretraining 对所有 token 计算 loss，这是两者的核心区别。

**Q10：为什么 SFT 用 5K 条数据就接近了 BERT 用 53K 条的效果？**

> 预期答法：Qwen2-0.5B 在大规模语料上预训练，已经具备了中文语义理解能力和新闻分类的背景知识。SFT 只需要教它"用什么格式输出答案"，而不需要从头学习语言特征。相当于 BERT 需要从头建立类别概念，而 Qwen2 只需要建立 prompt → 类别名 的映射，所需样本更少。这就是大模型"预训练 + 微调"范式的核心优势。
